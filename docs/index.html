About

PIE-V, Psychologically Inspired Error injection for Videos, constructs controlled mistake-aware egocentric procedural videos from clean Ego-Exo4D keystep procedures. The goal is to create examples where mistakes are not random corruptions, but human-plausible deviations with explicit procedural consequences and, when appropriate, recoveries.

The project contributes a structured error planner, a correction planner, LLM-based procedure rewriting and judging, video-editing case studies, and a human evaluation rubric for comparing generated mistake traces with existing egocentric mistake datasets.

From Clean Keysteps To Mistake-Aware Videos

The public repository is a demonstration of the PIE-V pipeline and evaluation protocol, not a redistribution of the full Ego-Exo4D video dataset. Source videos and original Ego-Exo4D annotations remain local inputs governed by the Ego-Exo4D access terms.

Ego-Exo4D Subset

PIE-V starts from 17 Ego-Exo4D keystep tasks, covering COVID test procedures, bicycle repair, CPR, and cooking tasks such as salads, noodles, omelets, tea, pasta, coffee latte, and sushi rolls. The subset contains 50 takes: three videos per task, except Cooking Sushi Rolls, where only one take is available, and COVID Test, where a fourth take is added because it is the most frequent task. The released manifest lists the 50 take names.

Structured Error Planning

The planner samples 102 mistakes across the 50 takes. Sampling is conditioned on video duration, step complexity, overall task complexity, procedure phase, working-memory load, and semantic step information from semantic role labeling. The repository includes the semantic resources and generated text examples used to inspect the resulting plans.

Recovery Mechanism

PIE-V adds 27 sampled corrections, so mistake-aware traces can represent recovery rather than only failure. Corrections are sampled conditionally on detectability, action type, latency, and repair strategy, including stopping to fix an action, rolling back and redoing a step, undoing an extra step, or performing a missed step later.

Human Evaluation

Five annotators evaluated the resulting scenarios with a nine-part rubric designed for plausible erroneous procedures. The repository provides the aggregate annotation summary and the annotation template used to structure the ratings.

Error Taxonomy + Correction

PIE-V uses a compact taxonomy that can map both generated mistakes and existing mistake datasets into a shared procedural language.

Wrong execution A step is attempted with the wrong object, side, amount, location, orientation, or manner.
Deletion A required step is skipped, creating a missing prerequisite or missing state change.
Substitution A plausible but incorrect alternative step replaces the intended step.
Insertion An unnecessary extra step is added, often through repetition or routine overgeneralization.
Transposition Two steps are performed in the wrong order while remaining locally plausible.
Correction A recovery step is explicitly tied to the mistake it detects and repairs.

Human Rubric

The rubric separates step-level plausibility from procedure-level coherence, so generated mistakes can be judged as complete procedural traces rather than isolated anomalies.

Metric Scale What it measures
Error Validity Binary Whether the deviation is a consequential mistake rather than a benign execution variant.
Human/Text Plausibility Likert 1-5 Whether a real person could naturally make this mistake in the given context.
Confusability Likert 1-5 How difficult the mistake is to notice during real-time execution.
Procedure Logic Binary + confidence 1-3 Whether the whole procedure becomes logically broken, weighted by annotator confidence.
Sequence Consistency Likert 1-5 Whether the edited step order remains executable as a coherent procedure.
State-Change Coherence Binary Whether object states and preconditions remain possible across the trace.
Taxonomy Fit Categorical Which error type or correction best explains the marked deviation.
Video Plausibility Likert 1-5 Whether the visual depiction looks natural when generated or annotated video is available.
Text-Video Grounding Likert 1-5 Whether the full textual trace matches the video at the episode level.

Repository Materials

The repository includes code for the planner, writer, judge, and analysis utilities; public semantic resources; generated plans and judged text examples; the 50-take manifest; aggregate annotation summaries; an annotator template; and anonymized video-generation scripts for representative case studies.

The project page stores the GIF previews directly in docs/assets. The MP4 links in the example captions point to the cvprw26 GitHub release assets, so they become live when that release is created and the MP4 files are uploaded there.
